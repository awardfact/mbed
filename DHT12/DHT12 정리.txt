I2C통신
데이터를 주고받는 선인(SDA)와 송수신 타이밍 동기를 위한 쿨럭 선(SCL)로 이루어진다 
하나의 마스터와 하나 이상의 슬레이브로 이루어지며 7비트여서 슬레이브 하나의 마스터에 최대 127개까지 연결할 수 있다
통신을 위해서는 SDA선과 SCL선이 모두 기본 HIGH상태여야 하기 때문에 
풀업 저항으로 HIGH상태를 만들어주는 것이 중요하다
데이터 송수신은 마스터에서 주도하고 데이터 송수신 전에 슬레이브를 주소를 명시해줘야 한다
I2C통신의 장점은 동기화 방식의 통신 방식이라는 점이다 쿨럭 신호를 이용해서 데이터 전송 타이밍을 맞추기 떄문에
통신 속도가 따로 정해지지 않아도 된다

통신은 시작신호와 데이터 신호,  정지 신호로 이루어진다 
SDA와 SCL는 기본적으로 HIGH상태이고 SDA신호만 LOW로 떨어지면 시작 신호로 판단한다 
그 다음에 SCL선으로 쿨럭 신호가 만들어지는데 쿨럭 신호가 LOW일떄가 SDA신호를 비트 신호로 바꾸는 시간
쿨럭 신호가 HIGH일 떄가 SDA신호를 읽는 시간이다 다시 SCL신호가 LOW가 되면 다음 비트 신호로 바꾸고 반복한다

한 쿨럭에 한 비트씩 데이터 신호를 만들고 모든 비트 전송이 끝나고 SCL신호가 HIGH가 되면 SDA신호도 HIGH로 만들어
정지 신호를 만든다
시작 신호 뒤에 나오는 첫 7비트는 슬레이브의 주소값이어야 하고 8번쨰 비트는 데이터를 읽어오기 위한 신호인지
쓰기위한 신호인지 나타내는 비트이다
슬레이브의 주소 값과 읽기/쓰기 비트는 마스터에서 생성할 수 있고 쓰기인 경우 마스터에서 데이터를 생성
읽기의 경우 슬레이브에서 데이터를 생성한다 8비트 데이터 전송 후 슬레이브에서 응답 신호를 만들어 수신을 확인한다
응답신호는 기본적으로 LOW여야 하고 슬레이브가 데이터를 전송하는 상태에서 모든 전송이 끝났을 때는 HIGH상태가 된다
이를 NACK이라고 하는데 이 외의 경우에 NACK가 발생하면 오류이다 



DHT12 
온도 , 습도센서 

핀
1.  VDD   - 전원공급장치 (2.7V~ 5.5V)
2.  SDA  -  시리얼 데이터 , 양방향
3.  GND  - 접지
4.  SCL  - 직렬 쿨럭 입력(단일 버스 접지)


직렬 쿨럭 입력(SCL)
SCL핀은 통신수단으로 사용된다 그리고  I2C를 쿨럭 라인이다
싱글 버스 선택을 나타내는 낮은 전원상태를 유지하고 통신을 한다
I2C통신에서는 통신 모드를 선택 하면 전원은 센서 통신중에 동일하게 유지된다 변경을 원하면
같은 방법을 사용하고 전원을 다시 켜야한다 그리고 통신 요구사항에 따라야 한다 
I2C통신을 선택하면 SCL은 동기화를 해야한다 
 
시리얼 데이터(SDA)
SDA핀은 데이터 일고 쓰기에 사용되는 3가지 상태 구조이다 특정 통신 타이밍의 상세는 상세 설명은 보면된다

I2C와 싱글 버스 통신 프로토콜
DHT12시리얼 인터페이스는 센서 신호와 전력 손실을 읽고 싱글 버스 센서와 함께 최적화한다 
I2C 양방향 출력 , 통신 모드 스위치는 버스에서 SCL 시그널 파워 케이블로 통신 모드를 결정하떄 사용하기 쉽다
전원을 켤때 SCL은 I2C와 높은 통신 모드 수준을 유지하기 위해 
SCL은 싱글 버스 통신 모드가 낮게 남아있다 
버스 통신에서 하나의 메인 라인은 I2C통신을 따르는 회사의 다른 상품과  완벽하게 호환된다 
주소는 I2C에 추가의 배선 장치 없이 직접 연결될 수 있다 
조작은 매우 간단하다 DHT12 센서를 읽으면 된다 
두 타입, 통신 프로토콜 그리고 타이밍은 엄격하게 따라야 한다 
특히 싱글 버스 통신 프로토콜 그리고 I2C통신 프로토콜을 상세하게 봐야한다 

DHT12 센서 I2C 통신 프로토콜
DHT12는 I2C통신 방식을 제공한다 I2C 기본 프로토콜이 준비되면
 I2C버스센서인 SDA 핀 I2C데이터 버스와 직접 연결할 수 있다
SCL은 I2C쿨럭 버스이고 사용자는 두 핀에 1k옴~10k옴 저항을 줘서 레지스터를 고정시켜야 한다 
I2C통신 속도는 400KHZ를 넘을 수 없다
